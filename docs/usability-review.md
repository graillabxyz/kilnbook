# Usability Review

- Run UX audits against `docs/ux-psychographics.md` before changing composer, profile, feed, firing, glaze, clay body, or privacy flows.
- Home is a ceramic feed, not an analytics dashboard.
- Profile setup starts with an identity label, then lets the profile add whichever record types apply.
- Live firing mode uses large touch targets for quick temperature, note, atmosphere, and photo actions.
- Firing records keep schedule, readings, environment, and result tagging in one workflow.
- Glazes and clay bodies are presented as canonical libraries with connected results.
- Posts are social wrappers; broad context links are optional, and precise visual evidence belongs in per-image result groups.
- Paid analytics are labeled as previews or gated by entitlement decisions.
- Mobile navigation prioritizes Home, Explore, Add, and Profile.
- Empty states explain the next action without substituting unrelated feed content.

Recommended next usability tests:

- Record a cone 6 electric firing using 30-minute readings.
- Record a gas reduction firing with atmosphere changes and wind notes.
- Upload two result images and tag different glazes and clay bodies on each.
- Open a glaze page and verify the correct authorized result appears automatically.
